Overview
What does this robot actually do?

Before touching anything, let's understand the full journey of one Instagram post through the system — start to finish.

The problem this solves: Manually visiting dozens of Instagram accounts, reading every post, deciding what's a real event, copying details, checking for duplicates, creating venues and organizers, and typing it all into Airtable — for every city, multiple times a week.

What the robot does instead: Three times a week, it wakes up on its own, checks every account on your list, downloads the flyer images, sends them to Gemini AI to read like a human would, pulls out every event detail, checks for duplicates, creates the venue and organizer records automatically if needed, saves the finished event to Airtable, and creates a WordPress draft. You only review and publish.

The complete flow, start to finish:
Schedule Trigger → Get Resources → Loop Accounts → Apify Instagram Scrape → Verify Posts
→ Loop Posts → Download Flyer → If Image OK → Resize → Rename for SEO
→ Prepare for Gemini Vision → Wait 30s → Gemini AI Agent → Parse Output → If Is Event
→ Check Duplicate → If Not Duplicate → Search/Create Venue → Search/Create Organizer
→ Format Event Name → Create Event in Airtable → Prep WordPress Payload → Upload Image
→ Merge → Attach Image → Create WordPress Draft → Update "Checked This Week" → Loop Back
✅
Why this solves the problem completely: Every manual task you described has its own dedicated node — visiting Instagram, reading flyers, checking duplicates, creating venues and organizers, formatting names, creating WordPress drafts. Nothing is left out, and nothing depends on you remembering to do it.
Step 1
Get your Airtable API key (and the other 3 keys)

Your Airtable tables are already built. The only thing left to do here is connect each service to n8n using an API key — think of an API key as a password that lets one app talk to another.

🔧 Airtable
  1. Go to airtable.com/create/tokens
  2. Click "Create new token"
  3. Give it any name, e.g. n8n - Jet Set Pets
  4. Under Scopes, add: data.records:read, data.records:write, and schema.bases:read
  5. Under Access, select your Jet Set Pets Events base
  6. Click Create token and copy it immediately — Airtable only shows it once
  7. In n8n, click any Airtable node → credential dropdown → "Create new credential" → paste the token → Save
🤖 Google Gemini
  1. Go to aistudio.google.com/apikey
  2. Click "Get API Key" → "Create API key"
  3. Copy the key
  4. In n8n, click the Google Gemini Chat Model node → credential dropdown → paste the key → Save
🕷️ Apify

⚠️ Do this BEFORE copying your API token

  1. Go to apify.com → search "Instagram Post Scraper" → click "Save to My Actors"
  2. Open the saved actor → click the Input tab
  3. You'll see a default Instagram URL already filled in — delete it (the field can't be left empty, so type any placeholder, e.g. https://www.instagram.com/instagram)
  4. Click "Save" only — do NOT click "Save & Run", this uses your credits for nothing
  1. Go to console.apify.com → Settings → Integrations
  2. Copy your API token
  3. In n8n, click the "Apify - Scrape Instagram" node → credential dropdown (Header Auth) → paste as Bearer YOUR_TOKEN → Save
🌐 WordPress
  1. Log in to your WordPress admin dashboard
  2. Go to Users → Profile → scroll to Application Passwords
  3. Type a name like n8n → click Add New Application Password
  4. Copy the password shown (you can only see it once!)
  5. In n8n, click any WordPress node → credential dropdown → enter your WordPress username + the application password → Save
🔧
One credential, many nodes. You only need to create each credential once per service. Every node that uses that service (e.g. all the Airtable nodes) will automatically share the same saved credential — you just select it from the dropdown.
Step 2
Every node in the workflow — explained simply

A "node" is one step in the assembly line. Click any node to read what it does in plain English. They run in order, like a relay race — each one hands off to the next.

✅
No need to run nodes one by one after testing. This workflow has already been tested. Once your 4 credentials are connected, just click "Execute Workflow" and the whole thing runs start to finish automatically. Node-by-node testing is only useful the first time, to confirm each credential works.
1
Schedule Trigger (Mon/Wed/Fri 9AM)
Trigger
▼

This is the alarm clock that starts the entire workflow. It wakes everything up every Monday, Wednesday, and Friday at exactly 9:00 AM. It produces no data of its own — it just tells the workflow "go!"

💭 You can change the days or time anytime by opening this node and adjusting the schedule settings.
2
Airtable - Get Active Resources
Database
▼

This node opens your Event Resources table and fetches every Instagram account that still needs to be checked this week. It filters using one condition: "Checked This Week" = uncheck.

It returns the Resource ID, Instagram URL, and Name for each matching account.

💭 Only accounts marked "uncheck" come through. Once an account is processed later in the workflow, it gets flipped to "checked" so it isn't picked up again until the next cycle resets it.
3
Split In Batches
Loop
▼

If you have 30 Instagram accounts to check, sending all 30 to Apify at once would overwhelm it. This node processes accounts one at a time — Account 1, then Account 2, then Account 3 — instead of all at once.

💭 Each account goes all the way through the whole workflow before the next one starts.
4
Apify - Scrape Instagram
Scraper
▼

This node visits the Instagram account and pulls the 5 latest posts. For each post it collects: caption, image URLs, carousel images (if it's a multi-photo post), timestamp, location tag, the account owner's username, whether the owner is verified, and any tagged users.

💭 Apify uses residential proxies so it looks like a real person browsing Instagram, which is why it can reliably read posts without getting blocked.
5
Verified_event
Code Filter
▼

A quality filter. It checks each post for one of two things: the account owner is verified (Instagram's blue checkmark), OR a tagged user in the post is verified. Only posts that pass one of these checks move forward.

💭 If zero posts pass, this node outputs a flag (_no_verified_posts: true) which the next gate uses to skip this account entirely and save resources.
6
Split Posts In Batches
Loop
▼

Same idea as Node 3, but now for individual posts. If an account has 5 verified posts, this node processes Post 1, then Post 2, then Post 3 — one at a time — instead of mixing them together.

This node has two paths: the main branch continues to "Download Flyer Image" to process the post, while the finish branch goes to "Update Resource Last Checked" once every post for this account is done.

💭 Processing one post at a time keeps the AI accurate — it never gets confused by mixing details from two different events.
7
Download Flyer Image
Image
▼

Apify only gives a web link to the image. This node actually downloads the image file from Instagram so it can be resized, renamed, and analyzed in later steps. Without this, the AI has nothing to look at.

💭 If the download fails (Instagram links sometimes expire), the workflow won't crash — it's set to continue regardless and move on.
8
If Image Downloaded
Gate
▼

A safety check. It looks at whether a binary file actually exists. If yes, the post continues into image processing. If no (the download failed or returned nothing), this post is skipped and the workflow moves to the next one.

9
Resize Image (650px shortest side)
Image
▼

Shrinks the image so its shortest side is 650 pixels. This makes the AI step faster, uses fewer AI tokens (which costs less), and prevents memory issues from huge image files.

10
Rename Image for SEO
Code
▼

Instagram images come with random file names. This code node renames the image to something descriptive and search-friendly, e.g. dog-friendly-event-boris-horton-july-12-2026-x7ab.jpg. The unique random code at the end guarantees the filename is never duplicated.

💭 Google reads image file names as part of its ranking. A descriptive name helps Jet Set Pets show up in searches like "dog-friendly events NYC."
11
Prepare Image for Gemini Vision
AI Prep
▼

Packages everything the AI will need into one bundle: the main flyer URL, any carousel (multi-photo) image URLs, how many carousel slides there are, the file name, and the file type. Think of it as putting the photo and a note together into one envelope before sending it to the AI.

12
Wait
Pause
▼

Pauses the workflow for 30 seconds before calling the AI. This exists purely to prevent a "429 Too Many Requests" error from Gemini — if posts are sent too quickly back to back, Gemini's servers will reject the requests. This short pause keeps everything running smoothly.

💭 Without this pause, a busy run with many posts could hit Gemini's rate limit and start failing partway through.
13
🧠 AI Agent - Extract Event Details
AI Agent ★ CORE
▼

This is the brain of the entire workflow, powered by the connected Google Gemini Chat Model (Gemini 2.5 Flash). It receives the flyer image, any carousel slides, the Instagram caption, and the location tag — and reads all of it like a human would.

Here's what it does and the rules it follows:

  • Reads the flyer visually first: It transcribes every word of text printed on the image into a field called image_text_extracted. If there's genuinely no text on the image, it outputs exactly "NO TEXT DETECTED".
  • Flyer beats caption when they disagree: If the image shows a different date/time/venue than the caption text, it trusts the flyer graphic as the authoritative source.
  • Decides if it's a real event: Sets is_event to true only if it's announcing a specific future event — not a recap or generic photo.
  • Calculates relative dates: If a post says "this Saturday," it works out the exact date using the post's timestamp as the reference point.
  • Picks the city from an approved list only: It must match one of the pre-approved city names exactly (Brooklyn, New York, Jersey City, Tampa Bay, etc.) — it can never invent a new city name.
  • Flags human-only events: If dogs aren't allowed at the event (e.g. a charity gala), it sets is_human_only to true and adds a clear warning note to the description.
  • Confidence is tied to visual proof: If no text was found on the image, the confidence score is capped at 70 because the AI is relying only on the caption. A score of 80–100 is only allowed when the core details are physically visible on the flyer.
  • Extracts contact info and notes: Scans for a website, phone number, and any special instructions like "bring a mat" or "RSVP required."
💭 How does an AI "see" an image? Gemini is "multimodal" — it can process images and text at the same time. It reads the pixels in the flyer the same way your eyes would, recognizing printed letters and numbers (this is called OCR), and then understands what those words mean in context — for example, connecting "7 PM Saturday" with a photo of a dog cafe to conclude "this is a dog-friendly evening event."
14
Parse AI Output
Code
▼

Gemini replies with a block of text. This node converts that text into clean, structured data the rest of the workflow can use. It also attaches the original Instagram URL and the resource ID so we always know exactly where each event came from.

15
If Is Event
Gate
▼

Checks two things from the AI's answer: is_event = true AND confidence is 60 or higher. Both must be true to continue. If either fails, the post is dropped — keeping low-quality or non-event posts out of your database.

16
Airtable - Check Duplicate Event
Database
▼

Searches your Events table for a record with the same Event Name AND the same Start Date. If a match exists, this event is already in your system.

17
Evaluate Duplicate
Code
▼

Looks at the search result and creates a flag: _is_duplicate. If nothing was found → false (it's new). If a record was found → true (it already exists).

18
If (Duplicate Gate)
Gate
▼

Reads the _is_duplicate flag. False (not a duplicate) → continues to venue creation. True (duplicate) → skips straight back to Split Posts In Batches to process the next post.

19
Airtable - Search Venue
Database
▼

Searches your Venues table for a record matching the venue_name the AI found. This is the automated version of checking your venue list before adding a new event.

20
Evaluate Venue Found
Code
▼

Creates a flag (_venue_found) telling the next node whether the venue already exists or needs to be created.

21
Airtable - Create Venue
Database
▼

If the venue doesn't exist yet, this node creates it automatically — storing the Venue Name, State, Zip Code, Phone, Website, and Notes pulled from the AI's output.

22
Resolve Venue ID
Code
▼

Every Airtable record has a unique ID. This node grabs the correct venue's ID — whether it's an existing record or one just created — and stores it as _resolved_venue_id so the event can be linked to the right venue.

23–25
Airtable - Search/Create Organizer + Resolve Organizer ID
Database
▼

Mirrors Nodes 19–22, but for the organizer instead of the venue. Searches the Organizers table for organizer_name, creates a new record (storing name, website, phone, category) if it doesn't exist, then resolves the correct ID as _resolved_organizer_id.

26
Format Event Name
Code
▼

Builds the final, search-friendly event name in your exact format, e.g. "Puppy Yoga @ Boris and Horton - 07/12/2026, New York, NY". It also generates the event tag, city, and state fields needed for the Airtable record.

💭 Adding the date and city to every event name prevents duplicate-sounding titles, which helps each event page rank individually on Google.
27
Airtable - Create Event
Database
▼

Saves the complete record into your Events & Experiences table: start/end date, start/end time, description, notes, event type, confidence score, the linked venue ID, the linked organizer ID, and the flyer image. This is the moment the event officially exists in your database.

28–31
Prep WordPress Payload → Upload Image → Merge → Attach Image ID → Create Draft Event
WordPress
▼

Five nodes working as a team to publish the event as a draft on your website:

  • Prep WordPress Payload: Converts the Airtable data into the title, content, and excerpt format WordPress expects.
  • WordPress - Upload Image to Media: Uploads the resized, SEO-named flyer to your WordPress Media Library and returns a media_id.
  • Merge: Combines the event text data with the uploaded media ID into one package.
  • Attach WP Image ID: Sets the uploaded image as the featured_media for the post.
  • WordPress - Create Draft Event: Creates the post with status draft — never published automatically.
💭 It's always saved as a Draft on purpose, so you get the final say before anything goes live.
32–33
Update Resource Last Checked + Continue Loop
Loop
▼

Update Resource Last Checked: Flips "Checked This Week" from "uncheck" to "checked" for this Instagram account in the Event Resources table — marking it as processed.

Continue Loop: Sends the workflow back to "Split In Batches" to grab the next Instagram account in the queue, and the whole cycle repeats.

💭 This keeps going until every account in your list has been processed, then the workflow finishes and waits for the next scheduled run.
Deep Dive
How the AI actually "sees" Instagram images

This is the most impressive part of the system. Here's exactly what happens when Gemini reads an event flyer.

📸 What the AI receives:
  • 🖼️ The flyer image (and carousel slides, if any)
  • 📝 The Instagram caption text
  • 📍 The location tag and post timestamp
📤 What the AI sends back:
  • ✅ is_event + confidence score
  • 📅 start_date, start_time, end_time
  • 📍 venue_name, city, organizer info
  • 📝 description, note, event_type
🧠
Vision AI explained simply: Gemini 2.5 Flash is "multimodal," meaning it can process images and text at the same time. When it receives a flyer, it reads the pixels visually — the same way your eyes would — and recognizes any printed text. This is OCR (Optical Character Recognition) combined with understanding: it doesn't just read "7 PM Saturday," it understands that means an upcoming event.

Help
Common problems and how to fix them
❌ "Airtable node shows no results / zero accounts found"
Check that at least one row in Event Resources has "Checked This Week" set to "uncheck". If every row is already "checked," the filter returns zero rows until the next reset.
❌ "Apify returns empty results"
Confirm you saved the actor with a placeholder URL (not the original default) and clicked Save only — not Save & Run. Also check the Instagram username is spelled correctly and the account is public.
❌ "Gemini AI returns no event data or a 429 error"
A 429 error means too many requests were sent too quickly — the "Wait" node (30 seconds) should prevent this, but if it still happens, try increasing the wait time slightly. Also confirm your Gemini API key is connected correctly.
❌ "WordPress draft is not being created"
Check your WordPress Application Password is correct and that the Events Calendar plugin (or your events post type) is active on your site.
❌ "Red node errors mid-run"
Click the red node — the error message usually tells you exactly what's wrong. Most common causes: wrong credential selected, or an Instagram image link that expired before it could be downloaded.
