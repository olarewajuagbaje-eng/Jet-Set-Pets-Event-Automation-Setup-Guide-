Overview
What does this robot actually do?

Before we build anything, let's understand what we're building and why it solves your problem 100%.

The problem right now: You have to manually visit 50+ Instagram accounts, read each post, figure out if it's an event, copy all the details, check if the event is already in your database, find or create the venue and organizer, and type everything into Airtable. For 15 cities. Multiple times a week. This is hundreds of hours of copy-pasting.

What this robot does instead: Three times a week, it wakes up automatically, visits every Instagram account on your list, downloads the images, sends them to an AI that reads the flyers and posts like a human would, extracts all the event details, checks your database for duplicates, creates the venue and organizer if they don't exist, saves the event to Airtable, and creates a draft on your WordPress website. You just review and publish.

✅
Why this solves the problem 100%: Every single manual task you described — visiting Instagram, reading flyers, checking duplicates, creating venues, formatting event names, creating WordPress drafts — is handled by a specific node in this workflow. There is a node for each step. Nothing falls through the cracks.

Step 3
Every node in the workflow — explained simply

A "node" is like one step in the assembly line. Click any node below to read what it does in plain English. They run in order, like a relay race — each one passes the baton to the next.

✅
Good news — no need to run nodes one by one! This workflow has already been built and tested. Once you have added all your API keys (Step 2), you just click the big "Execute Workflow" button and the entire system runs automatically from start to finish. The node-by-node testing is only needed during your initial setup to check credentials — after that, just run the whole workflow at once.
1
Schedule Trigger
Trigger
▼

Think of this as an alarm clock for your robot. This is the very first node — it is what starts the entire workflow. It wakes up the automation every Monday, Wednesday, and Friday at exactly 9:00 AM. You don't have to press anything — once you turn the workflow on, it runs on this schedule forever, automatically.

💭 Why 3 days a week? Dog-friendly events are usually announced a few days in advance. Checking 3 times a week means you catch new posts quickly without spending too many Apify credits. You can change this schedule anytime by clicking this node and adjusting the time settings.
2
Airtable – Get Active Resources
Database
▼

This node opens your "Event Resources" table in Airtable and fetches only the Instagram accounts that are ready to be checked. It looks for two conditions at the same time:

  • "Is Active" is ticked ✓ — this account is active and should be checked
  • "Info Quality" is set to "Great" — this account regularly posts reliable event info

If an account is marked "Good" or "Check monthly", it gets skipped during this automatic run. You can still check those manually whenever you want.

💭 The "Great" + "Is Active" filter logic: Both conditions must be true for an account to be included. This is how you control which accounts the robot checks — just tick or untick "Is Active", or change the Info Quality rating, to include or exclude any account.
3
Split In Batches (First one — Account Batching)
Loop
▼

Imagine you have 50 Instagram accounts to check. If you tried to send all 50 to Apify at the exact same time, it would get overwhelmed and crash. This node organizes your accounts into a line so they are processed one at a time, in an orderly queue — like a queue at a coffee shop. One account gets processed completely before the next one starts.

💭 Think of it as the workflow's organizer — it makes sure everything runs smoothly without overloading any service.
4
Apify – Scrape Instagram
Scraper
▼

This is your Instagram reading robot. It takes each Instagram account URL from your list and visits that account to collect the most recent posts. For each post, it grabs three things: the caption text, the image URL (the link to the actual photo), and the date it was posted.

💭 Why Apify instead of going to Instagram directly? Instagram has rules that block automated tools from reading posts. Apify has special technology that makes it look like a real person browsing Instagram, so it works reliably without getting blocked.
⚠️ Remember: You must have already set up the Instagram Post Scraper actor in Apify (with a placeholder URL saved) before this node will work. See Step 2 above for that setup.
5
Verified_event (Code Filter)
Code Filter
▼

After Apify grabs the posts, this is a small quality check. It is a code node — meaning it runs a short, automatic script. It filters the posts to check whether the account or any tagged user has a verified Instagram badge (the blue checkmark ✓). This helps ensure you're only looking at reliable, legitimate accounts and not spam.

💭 If zero verified posts are found, this node creates a flag called _no_verified_posts: true which tells the next node to skip this account entirely and move on to the next one. This saves time and money.
6
If Has Verified Posts
Gate / Filter
▼

Think of this as a doorman at a club. It checks the flag from the previous node. If there are no verified posts, the doorman says "Not tonight" and the workflow skips this account completely and moves on to the next. If there ARE verified posts, everyone gets through and the workflow continues.

💭 This is a smart way to save resources. Why spend AI credits analyzing posts from an account that has nothing useful? Gates like this make the workflow efficient.
7
Split Posts In Batches (Second one — Post Batching)
Loop
▼

This node does the same job as Node 3, but now for individual posts instead of accounts. Once Apify returns several posts from one account, this node lines them up and processes them one post at a time. This makes the AI much more accurate because it focuses on one single post at a time instead of getting confused by multiple posts at once.

💭 After each post is fully processed all the way to the end (saved to Airtable and sent to WordPress), the workflow loops back here to pick up the next post in the queue.
8
Download Flyer Image
Image
▼

Apify only gives you a web link to the Instagram image — like a web address pointing to the photo. But the AI can't read a web address — it needs the actual image file. This node downloads the real image file from that link so it can be resized, renamed, and analyzed by the AI in the next steps.

💭 If the download fails — for example, Instagram sometimes makes post links expire — the workflow won't crash. It has a safety setting that lets it continue on to the next post automatically.
9
If Image Downloaded
Gate / Filter
▼

Another safety check — like a guardrail. This node asks: did the image actually download successfully? If yes, the workflow moves forward. If no (the download failed or the file is empty), the workflow skips this post and moves on to the next one. This prevents crashes caused by bad or expired image links.

10
Resize Image (650px)
Image
▼

Instagram images can be very large files. This node resizes the image to 650 pixels wide — a standard size that is large enough to look good on your website but small enough to not slow down your page load. It also ensures the image isn't too large for the AI to process quickly.

💭 Smaller images = faster website = better Google ranking. This is a small but important step.
11
Rename Image for SEO
Code
▼

By default, images from Instagram have random file names like IMG_38471.jpg. This code node gives each image a smart, descriptive name based on the venue, month, and year — for example: dog-friendly-event-boris-horton-june-2025.jpg. This helps your website rank higher on Google because Google reads image file names as part of its search ranking.

💭 This is called SEO (Search Engine Optimization). Simple changes like this help Jet Set Pets show up when people search for "dog-friendly events NYC" on Google.
12
Prepare Image for Gemini Vision
AI Prep
▼

The AI needs two pieces of information to work: the actual image file AND the Instagram caption text. This node packages both together into one bundle so they can be sent to the AI at the same time. Think of it like putting a letter and a photo into the same envelope before mailing it.

13
🧠 AI Agent – Extract Event Details (Gemini 2.5 Flash or OpenAI)
AI Agent ★ CORE
▼

This is the brain of the entire workflow. This is where the magic happens. The AI Agent receives the image and the caption text and does something remarkable: it reads the event flyer like a human would.

Here is exactly what the AI does:

  • Looks at the image: Gemini 2.5 Flash is a "vision AI" — meaning it can see and understand images, not just text. It reads all text printed on the flyer (dates, times, venue names, event names) just like your eyes would.
  • Reads the caption: It also reads the Instagram caption text to find any extra details not visible in the image.
  • Decides if it's an event: It asks itself: "Is this actually an upcoming event, or just a regular photo or ad?" Only if it is an event does it continue.
  • Gives a confidence score: It scores how confident it is (0 to 1). A score above 0.6 means it's likely an event.
  • Extracts the data: It pulls out and returns: event name, event date, start time, end time, venue name, organizer name, event website URL, and description.
🤖
Gemini vs OpenAI — how to swap: By default this workflow uses Gemini 2.5 Flash. If you want to use OpenAI GPT-4o instead, click this node → find the "Model" or "Chat Model" setting → change it to OpenAI → select GPT-4o. The rest of the node stays the same. Both AI models can read images and understand event details equally well.
💭 How does an AI "see" an image? AI models like Gemini and GPT-4o are trained on millions of images with labels describing what's in them. Over time, the AI learned to recognize patterns — letters, numbers, logos, layouts. When it sees a flyer image, it can recognize the text inside it (like "7 PM" or "Trivia Night") and understand what those words mean in context. This technology is called OCR + Vision AI.
14
Parse AI Output
Code
▼

The AI sends back its answer as a big block of text. This code node cleans up that response and converts it into a tidy, organized format (called JSON) that the rest of the workflow can read and use. Think of it like taking a messy handwritten note and typing it up neatly into a form.

15
If Is Event (Confidence Filter)
Gate / Filter
▼

Not every Instagram post is an event — some are just photos, promotions, or reshares. This gate checks two things from the AI's output:

  • is_event = true — the AI confirmed this is actually an event
  • confidence score ≥ 0.6 — the AI is at least 60% sure

If both conditions are met, the workflow continues. If not, this post is discarded — saving you from getting junk data in your Airtable.

16
Airtable – Check Duplicate Event
Database
▼

Before saving a new event, the workflow checks if you already have it. This node searches your "Events and Experiences" table to see if an event with the same name and date already exists. This is exactly what you do manually when you go to the Events tab in Airtable and scroll to check — except the robot does it in milliseconds.

17
Evaluate Duplicate
Code
▼

After searching Airtable, this code node makes the final yes/no decision: is this a duplicate or not? It looks at what the Airtable search returned. If nothing was found → it marks this event as "Not Duplicate" and allows it to continue. If something was found → it marks it as "Duplicate" so the next node can stop it.

18
If (Duplicate Check Gate)
Gate / Filter
▼

This is the final check before creating anything in Airtable. It reads the result from Node 17. If "Not Duplicate" → proceed to create the venue, organizer, and event. If "Duplicate" → stop and move on to the next post. This is how the robot makes sure you never get the same event in Airtable twice.

19
Airtable – Search Venue
Database
▼

The AI extracted a venue name from the event (for example, "Boris & Horton"). Before creating the event, the workflow needs to link it to a venue record. This node searches your Venues table to see if that venue already exists. This is the automated version of you manually checking your venues list.

💭 This is exactly how you do it manually — check if the venue is already in your list before adding a new event. The robot does it automatically.
20
Evaluate Venue Found
Code
▼

After searching for the venue, this code decides: was the venue found or not? If found → use the existing venue record. If not found → the next node will create a brand new venue. This result is passed along so the right action is taken.

21
Airtable – Create Venue (if new)
Database
▼

If the venue did not exist in your Venues table, this node automatically creates a new venue record using the information the AI found. So if the AI found "Mutts & Martinis Tampa" and it's not in your list yet, it gets added automatically — just like you would add it manually before creating the event.

22
Resolve Venue ID
Code
▼

Every record in Airtable has a unique ID code — something like recABC123xyz. When we link an event to a venue, we need that exact ID. This node grabs the correct venue ID — whether it came from an existing venue or a newly created one — so the event can be properly linked to it. Without this, the event and the venue would be unconnected in Airtable.

23–25
Airtable – Search / Create Organizer + Resolve Organizer ID
Database
▼

These three nodes do exactly the same thing as Nodes 19–22, but for the event organizer instead of the venue. The workflow searches the Organizers table for the organizer the AI found. If they don't exist yet, a new organizer record is created. Then the organizer's unique ID is grabbed so it can be linked to the event.

💭 This is why every event in your Airtable will always have both a venue AND an organizer properly linked — even if they're brand new ones the robot has never seen before.
26
Format Event Name
Code
▼

You have a specific naming format for events — it includes the date and city to prevent duplicate content problems on Google. For example: "Trivia Night at Boris & Horton – NYC – June 15 2025". This code node automatically formats the event name in exactly that way, every time, without you having to type it.

💭 Why does this matter for Google? If 10 events are all just named "Trivia Night", Google gets confused and may not show any of them. Adding the date and city makes each event page unique, which is better for SEO (search rankings).
27
Airtable – Create Event
Database
▼

This is the moment of truth! All the collected and formatted data — event name, start date, end date, start time, end time, description, venue ID, organizer ID, website link, image URL — gets saved as a brand new record in your "Events and Experiences" table in Airtable.

This is the automated version of you sitting down and manually typing all the event details into Airtable one row at a time. The robot does it in milliseconds, perfectly formatted, every time.

28–31
WordPress Pipeline: Prep → Upload Image → Merge → Create Draft
WordPress
▼

These four nodes work as a team to publish the event to your WordPress website as a draft. Here's what each one does in plain terms:

  • Prep WordPress Payload: Converts the Airtable event data into the exact format that WordPress understands. Like translating from one language to another.
  • WordPress – Upload Image to Media: Uploads the resized, SEO-renamed flyer image directly to your WordPress Media Library.
  • Merge: Combines the event text data and the freshly uploaded image ID into one complete package to send to WordPress.
  • Attach WP Image ID: Links the uploaded image to the event draft as the featured image.
  • WordPress – Create Draft Event: Creates the event inside your WordPress Events Calendar plugin as a Draft — not live yet, so you can review before publishing.
💭 Events are saved as Drafts on purpose. This gives you a chance to do a final quality check before anything goes live on your website. The robot does all the data entry; you do the final review and click Publish when you're happy with it.
32–33
Update Resource Last Checked + Continue Loop
Loop
▼

These are the final two nodes in each cycle of the loop:

  • Update Resource Last Checked: Goes back to the Event Resources table in Airtable and updates the "Last Checked" field for this Instagram account to today's date. This way you always know exactly when each account was last reviewed.
  • Continue Loop: Signals to the system that this post is done! The workflow goes back to the batch queue (Node 7 for the next post, or Node 3 for the next account) and repeats the entire process for the next item.
💭 This loop continues until every "Is Active" + "Great" Instagram account has been fully processed. Once all accounts are done, the workflow finishes quietly and waits until the next scheduled run (Mon/Wed/Fri 9AM).
Deep Dive
How the AI actually "sees" Instagram images

This is the most impressive part of the whole system. Here's exactly what happens when Gemini reads an event flyer.

📸 What the AI receives:
  • 🖼️ The actual image file (the flyer or post photo)
  • 📝 The Instagram caption text
  • 🔗 The original post URL
📤 What the AI sends back:
  • ✅ is_event: true or false
  • 🎯 confidence: 0.0 – 1.0 (how sure it is)
  • 📅 event_date, start_time, end_time
  • 📍 venue_name, organizer_name
  • 📝 description, website_url
🧠
Vision AI explained simply: Gemini 2.5 Flash is what's called a "multimodal" AI — "multimodal" just means it can handle more than one type of input at the same time (both text AND images). When you send it an event flyer image, it processes the pixels the same way your eyes do, and reads any text it sees on that image. This is called OCR (Optical Character Recognition) combined with AI understanding. So it doesn't just read the words — it understands what they mean. "7 PM Saturday" + a photo of a dog bar = "this is a dog-friendly evening event."

Help
Common problems and how to fix them

Something not working? Find your error below.

❌ "Airtable node shows no results / zero accounts found"
Check that at least one row in your Event Resources table has BOTH: "Is Active" ticked ✓ AND "Info Quality" set exactly to "Great". Both conditions must be true. If either is missing, the filter returns zero rows and the whole workflow stops early.
❌ "Apify returns empty results"
First, check that you did the Apify actor setup — open the actor in your Apify console, go to the Input tab, make sure there's a placeholder URL saved (NOT the default one Apify put in). Also verify the Instagram URL in your Airtable is the full profile URL (e.g. https://www.instagram.com/borisandhorton). Also check the account is public and exists.
❌ "AI agent returns no event data or weird results"
Check that your Gemini API key (or OpenAI key) is added correctly in n8n credentials. Also check that the image was downloaded successfully in Node 8 — if the image link expired, the AI has nothing to look at. This is normal for some posts.
❌ "Event Start Time / End Time is showing an error"
Remember these fields were changed to Single line text in Airtable. If you set them as Date or Time field types, the robot won't be able to write plain text like "7:00 PM" into them. Open Airtable → click the field name → change the field type to "Single line text".
❌ "WordPress draft is not being created"
Check your WordPress Application Password. Make sure the "The Events Calendar" plugin is installed and active on your WordPress site — the workflow uses that plugin's event post type. Also check that Pretty Permalinks are enabled: WordPress Admin → Settings → Permalinks → Post name → Save.
❌ "Red node errors — workflow stops partway through"
Click the red node. An error message will appear explaining exactly what went wrong. The most common causes are: wrong credential selected, wrong Base ID in an Airtable node, or an Instagram image link that expired. Read the error message carefully — it usually tells you exactly what to fix.
